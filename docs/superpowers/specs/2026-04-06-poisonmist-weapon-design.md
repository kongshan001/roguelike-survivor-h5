# 设计规格书：第8种基础武器 -- 毒雾 (PoisonMist)

> 版本: v2.1 | 策划 Agent | Drive #25
> 状态: 设计完成，待前端实现
> 前置依赖: 无（独立于幸运硬币/无尽模式）

## 1. 设计概述

当前7种基础武器覆盖了环绕(圣水)、投射(飞刀)、随机(闪电)、范围(圣经)、锥形(火焰法杖)、光环(冰冻光环)、追踪回旋(回旋镖)七种攻击模式。缺少**DOT区域型**武器 -- 以玩家为中心释放持续伤害毒雾区域，范围内敌人持续受到毒伤害。

毒雾的核心差异化特征：
- **纯DOT无减速**：与冰冻光环（减速+冰冻+低伤害）不同，毒雾是高伤害+无控制。两者是"伤害vs控场"的互补关系
- **毒叠层机制**：敌人在毒雾中停留越久，毒叠层越高，DPS递增。这鼓励玩家主动保持距离（让敌人在毒雾中停留更久）而非一味逃跑
- **视觉辨识度极高**：绿色毒雾+紫色气泡粒子，与所有现有武器视觉风格截然不同
- **Build空间扩展**：7->8种基础武器，进化组合从C(7,2)=21扩展到C(8,2)=28（+33%）

### 与现有武器的差异化对比

| 维度 | 圣经 | 冰冻光环 | 火焰法杖 | **毒雾(新)** |
|------|------|---------|---------|------------|
| 范围形状 | 圆形旋转 | 圆形光环 | 锥形 | **圆形(玩家中心)** |
| 控制效果 | 无 | 减速+冰冻 | 无 | **无** |
| 伤害类型 | 瞬间接触 | 低DPS | 接触+点燃DOT | **纯DOT叠层** |
| 玩家操作 | 走位控范围 | 被动 | 朝向瞄准 | **走位保持距离** |
| 定位 | 持续AOE | 控场 | 方向性AOE | **纯伤害AOE** |

## 3. 进化路线详细规格

### 3.1 进化路线A：毒雾 + 圣经 -> 腐蚀领域 (CorrosiveDomain)

**设计理念**：圣经的旋转范围特性 + 毒雾的DOT叠层 = 超大范围持续毒雾 + 旋转毒球。这是"最大范围持续伤害"的极致表达。

**核心机制**：
- 继承毒雾的DOT叠层系统，范围扩大到170px
- 新增3个旋转毒球（类似圣经的旋转），毒球接触敌人额外造成4点伤害
- 毒球旋转半径140px，旋转速度3 rad/s
- 叠层上限提升到6，叠层更快（0.5秒/层）

**数值对比**：

| 属性 | 毒雾Lv3 | 腐蚀领域 | 提升 |
|------|---------|---------|------|
| 半径 | 130px | 170px | +31% |
| 基础DPS | 2.5 | 3.5 | +40% |
| 每叠层DPS | 1.2 | 1.5 | +25% |
| 最大叠层 | 5 | 6 | +20% |
| 满叠层总DPS | 2.5+5x1.2=8.5 | 3.5+6x1.5=12.5 | +47% |
| 毒球DPS | - | ~12(3球x4) | 新增 |
| **总DPS(满叠层+毒球)** | **8.5** | **~24.5** | **+188%** |

**视觉**：
- 深绿色毒雾范围（`rgba(76,175,80,0.15)` 渐变到透明）
- 3个深紫色毒球（`#7b1fa2`，10x10px）在范围内旋转
- 毒球拖尾紫色粒子
- 范围边缘有绿色气泡上浮

### 3.2 进化路线B：毒雾 + 磁铁 -> 瘟疫磁力 (PlagueMagnet)

**设计理念**：磁铁的拾取范围特性 + 毒雾的范围 = 毒雾范围变成"拾取真空区"。这创造了一个有趣的策略：让敌人进入毒雾 = 被毒死 + 掉落的宝石自动飞向玩家。是"经济+战斗"的混合进化。

**核心机制**：
- 继承毒雾的DOT叠层系统（与Lv3完全相同）
- 新增：毒雾范围内的宝石自动飞向玩家（吸引范围50px，速度180px/s）
- 不影响毒雾外的宝石（仅范围内的宝石被吸引）

**数值对比**：

| 属性 | 毒雾Lv3 | 瘟疫磁力 | 提升 |
|------|---------|---------|------|
| 半径 | 130px | 130px | 不变 |
| 基础DPS | 2.5 | 2.5 | 不变 |
| 满叠层总DPS | 8.5 | 8.5 | 不变 |
| 宝石吸引 | 无 | 范围内50px | **新增** |

**设计理由**：
- 战斗数值不变，增加"品质生活"维度 -- 不需要走过去捡宝石
- 与磁铁被动叠加：磁铁扩大基础拾取范围35px，瘟疫磁力额外吸引毒雾范围内宝石50px
- 适合"站桩流"Build：站在原地毒杀 + 宝石自动飞来 + 经验自动获取

**视觉**：
- 毒雾范围边缘有绿色磁力线效果（旋转的细线）
- 宝石被吸引时有绿色拖尾

## 4. 毒叠层系统详细规格

### 4.1 叠层机制

毒雾使用独立的 `_poison` 标记系统（与火焰法杖的 `_burn` 类似但独立）：

```
敌人数据结构新增:
e._poison = { stacks: 0, timer: 0, dotTimer: 0 }
// stacks: 当前毒叠层 (0-maxStacks)
// timer: 叠层检查计时器
// dotTimer: DOT伤害计时器（用于计算离开后的衰减）
```

**叠层逻辑流程**：

1. **在毒雾范围内**：
   - 每 `stackInterval` 秒检查一次，stacks += 1（上限 maxStacks）
   - 每秒造成 (baseDps + stacks * stackDps) 的伤害
   - `dotDuration` 重置（保持在范围内时不衰减）

2. **离开毒雾范围后**：
   - stacks 不立即清零，而是在 `dotDuration` 秒内线性衰减
   - 衰减期间仍造成 (baseDps + stacks * stackDps * 衰减比例) 的伤害
   - dotDuration 归零后 stacks = 0，停止造成伤害

3. **重新进入毒雾范围**：
   - stacks 从当前值继续叠加（不重置为0）
   - dotDuration 重置

### 4.2 伤害公式

```
每秒伤害 = baseDps + currentStacks * stackDps
实际伤害/帧 = (baseDps + currentStacks * stackDps) * dt
```

**满叠层DPS表**：

| 等级 | 基础DPS | 每层DPS | 最大叠层 | 满叠层总DPS |
|------|---------|---------|---------|------------|
| Lv1 | 1.5 | 0.8 | 3 | 1.5+3x0.8 = 3.9 |
| Lv2 | 2.0 | 1.0 | 4 | 2.0+4x1.0 = 6.0 |
| Lv3 | 2.5 | 1.2 | 5 | 2.5+5x1.2 = 8.5 |
| 腐蚀领域 | 3.5 | 1.5 | 6 | 3.5+6x1.5 = 12.5 (+毒球12 = 24.5) |
| 瘟疫磁力 | 2.5 | 1.2 | 5 | 2.5+5x1.2 = 8.5 |

### 4.3 与其他系统的交互

**与火焰法杖点燃 `_burn` 的交互**：
- 毒和火是独立的DOT系统，两者可以同时存在
- 敌人可以同时 `_burn` 和 `_poison`，两种伤害叠加计算
- 这鼓励"火毒混合Build"（火焰法杖+毒雾）- 双重DOT

**与冰冻光环的交互**：
- 冰冻的减速让敌人在毒雾中停留更久 = 更多叠层 = 更高DPS
- 天然协同：冰冻光环减速 -> 毒雾高DOT -> 冰毒控杀流

**与连击系统的交互**：
- DOT伤害击杀计入连击（与火焰法杖点燃击杀一致）
- 毒雾的高击杀频率有利于维持连击

**与幸运硬币的交互**：
- 剧毒炼金协同：毒杀敌人额外掉落1宝石
- 毒雾击杀频率高，剧毒炼金收益显著

## 5. Synergy 协同详细规格

### 5.1 poisonmist_armor: 猛毒护甲

| 属性 | 数值 |
|------|------|
| 条件 | 毒雾 + 护甲 |
| 效果 | 毒雾范围内受伤减少20% |
| 定位 | 防御增强型协同 |

**设计理由**：
- 毒雾以玩家为中心，意味着玩家始终在毒雾范围内
- "站在毒雾中更安全"创造了一个有趣的战术选择：主动进入毒雾区域
- 与护甲的减伤乘法叠加：实际受伤 = 基础伤害 x (1-护甲) x (1-0.2)
- 不改变毒雾的伤害输出，纯防御增强

**实现要点**：
- 在 Player.takeDamage() 中检查：`if (hasSynergy('poisonmist_armor') && isInPoisonMist) dmg *= 0.8`
- 需要一个方法检测玩家是否在毒雾范围内（因为毒雾跟随玩家，所以拥有毒雾时始终为 true）
- 简化为：`if (hasSynergy('poisonmist_armor') && player has poisonmist weapon) dmg *= 0.8`

### 5.2 poisonmist_regen: 腐蚀再生

| 属性 | 数值 |
|------|------|
| 条件 | 毒雾 + 再生护符 |
| 效果 | 毒雾范围内再生速度翻倍 |
| 定位 | 续航增强型协同 |

**设计理由**：
- 与 FrostAura 的 frost_regen 协同（冰冻概率+5%/s）形成对比
- 毒雾以玩家为中心，玩家始终在毒雾范围内 = 再生始终翻倍
- 与猛毒护甲协同叠加：更硬 + 回血更快 = 站桩流核心
- 再生护符Lv3=每3秒1HP，翻倍=每1.5秒1HP，在高密度敌人环境中提供可观续航

**实现要点**：
- 在再生计时器逻辑中检查：`if (hasSynergy('poisonmist_regen') && hasWeapon('poisonmist')) regenInterval *= 0.5`
- 与 boots_regen（移动时再生翻倍）不叠加（两者取最大效果）

### 5.3 poisonmist_luckycoin: 剧毒炼金

| 属性 | 数值 |
|------|------|
| 条件 | 毒雾 + 幸运硬币 |
| 效果 | 毒杀敌人额外掉落1宝石(价值+1) |
| 定位 | 经济增强型协同 |

**设计理由**：
- 毒雾是高击杀频率武器（满叠层8.5DPS，范围130px），击杀数多
- 与 holywater_luckycoin（圣水击杀额外1金币）形成互补 -- 一个加金币，一个加宝石
- 宝石价值+1 = 经验+1 + 金币间接+1（通过 gemToGold）
- 不影响战斗平衡，纯经济/经验增强

**实现要点**：
- 在毒DOT击杀时检查：`if (hasSynergy('poisonmist_luckycoin') && killFromPoison) gemValue += 1`
- "毒杀"定义：敌人当前有 `_poison` 且 stacks > 0 时被击杀
- 与 firestaff_luckycoin（点燃敌人宝石+1）类似机制，可复用

## 8. 视觉设计规范

### 8.1 毒雾本体（PoisonMist）

**范围绘制**：
- 以玩家为中心的圆形半透明绿色区域
- 外圈：`rgba(76,175,80,0.12)` 渐变到透明
- 内圈：`rgba(56,142,60,0.08)` 更淡
- 脉动效果：范围边缘每2秒一次绿色脉冲

**气泡粒子**：
- 范围内随机生成绿色气泡（3x3px `#81c784`），向上漂浮

## 9. 数值平衡分析

### 9.1 DPS对比表（满叠层状态）

| 武器 | Lv1 DPS | Lv3 DPS | 进化后DPS | 定位 |
|------|---------|---------|----------|------|
| Knife | ~3 | ~8 | ~25(FireKnife) | 高频低伤单体 |
| HolyWater | ~3 | ~8 | ~15(ThunderHolyWater) | 环绕持续 |
| Lightning | ~2.5 | ~6 | - | 随机链式 |
| Bible | ~3 | ~7 | ~15(HolyDomain) | 范围持续 |
| FireStaff | ~3 | ~7+DOT | - | 锥形+点燃 |
| FrostAura | ~1 | ~2 | ~10(Blizzard) | 控场 |
| Boomerang | ~6 | ~20 | ~24(Thunderang) | 单体之王 |
| **PoisonMist** | **~2.3** | **~8.5** | **~24.5(CorrosiveDomain)** | **范围DOT之王** |

**Lv1 DPS计算**：1.5(基础) = 1.5, 1层后 = 1.5+0.8 = 2.3, 3层满 = 1.5+3x0.8 = 3.9
**Lv3 DPS计算**：满叠层 = 2.5+5x1.2 = 8.5

**平衡结论**：
- Lv1 DPS 2.3 低于飞刀(3)和圣经(3)，但满叠层后 3.9 超过冰冻光环(1)。合理 -- 前期弱但成长高
- Lv3 满叠层 8.5 与圣经Lv3(7)、圣水Lv3(8)相当。合理 -- 同为AOE武器
- 腐蚀领域 24.5 是所有进化武器中最高的，但需要敌人在范围内停留5秒以上才能达到满叠层。实际平均DPS约15-18，与FireKnife(25)、Thunderang(24)相近

### 9.2 叠层时间分析

| 等级 | 叠层间隔 | 达到满叠层时间 | 满叠层持续时间(离开后) |
|------|---------|--------------|---------------------|
| Lv1 | 1.0s | 3.0s | 2.5s |
| Lv2 | 0.8s | 3.2s | 3.0s |
| Lv3 | 0.6s | 3.0s | 3.5s |
| 腐蚀领域 | 0.5s | 3.0s | 4.0s |

**关键发现**：所有等级达到满叠层都需要约3秒，这创造了一个一致的"3秒融化"节奏。玩家需要让敌人在毒雾中停留至少3秒才能达到最高DPS。

### 9.3 与5分钟短局的适配性

- 毒雾是被动型武器（跟随玩家，不需要瞄准），与其他武器同时装备不增加操作负担
- 满叠层需要3秒，5分钟内敌人通常在毒雾中停留1-5秒（取决于敌人移速）
- 对慢速敌人（僵尸40px/s、骷髅20px/s）非常有效 -- 它们在毒雾中停留时间长
- 对快速敌人（蝙蝠80px/s）效果较差 -- 它们快速穿过毒雾，叠层低
- 这创造了一个自然的"毒雾克制慢敌人、飞刀克制快敌人"的互补关系

**PoisonMist 类核心实现**：

```js
// 伪代码
export class PoisonMist extends Weapon {
  constructor(owner) { super('poisonmist', owner); this.pulseTimer = 0; this.particles = []; }
  get radius() { /* 从 CFG.POISONMIST.base.levels[this.level] 读取 */ }
  get baseDps() { /* 同上 */ }
  // ... 其他 getter

  update(dt, enemies, sfx) {
    this.pulseTimer += dt;
    const cfg = CFG.POISONMIST.base.levels[this.level];
    for (const e of enemies) {
      const d = dist(this.owner, e);
      if (d < cfg.radius) {
        // 初始化毒标记
        if (!e._poison) e._poison = { stacks: 0, timer: 0, dotTimer: cfg.dotDuration };
        // 叠层计时
        e._poison.timer += dt;
        if (e._poison.timer >= cfg.stackInterval && e._poison.stacks < cfg.maxStacks) {
          e._poison.stacks++;
          e._poison.timer = 0;
        }
        // DOT伤害
        const dps = cfg.baseDps + e._poison.stacks * cfg.stackDps;
        e.hurt(this.applyDmg(dps * dt));
        // 重置离开计时器
        e._poison.dotTimer = cfg.dotDuration;
      }
    }
    // 处理不在范围内的敌人的毒衰减
    for (const e of enemies) {
      if (e._poison && e._poison.dotTimer > 0) {
        const d = dist(this.owner, e);
        if (d >= cfg.radius) {
          e._poison.dotTimer -= dt;
          // 衰减期间仍造成伤害
          const decayFactor = e._poison.dotTimer / cfg.dotDuration;
          const dps = (cfg.baseDps + e._poison.stacks * cfg.stackDps) * decayFactor;
          e.hurt(this.applyDmg(dps * dt));
          // 叠层线性衰减
          e._poison.stacks = Math.max(0, e._poison.stacks - dt / cfg.dotDuration * cfg.maxStacks);
          if (e._poison.dotTimer <= 0) e._poison = null;
        }
      }
    }
  }

  draw(ctx, cam, canvas) { /* 见视觉规范第8节 */ }
}
```

## 13. 设计决策记录

1. **选择毒雾而非光束/召唤**：
   - 光束(LaserBeam)实现复杂度高（射线检测+连续伤害+反射等），需要新的碰撞系统
   - 召唤(Summon)需要AI宠物系统，开发量大且与移动端虚拟摇杆操作冲突
   - 毒雾是"范围DOT"型，复用现有的圆形范围判定（冰冻光环）+ DOT逻辑（火焰法杖点燃），实现复杂度最低
   - 竞品对标：VS的Poison/Druid和Halls of Torment的DOT Build都证明此类型有效

2. **叠层机制而非固定DPS**：
   - 固定DPS（如冰冻光环的2/s）太无聊，缺乏成长感
   - 叠层机制创造"越打越痛"的正反馈循环，与竞品中20 Minutes Till Dawn的Synergy发现感类似
   - 3秒满叠层的节奏与5分钟短局匹配 -- 不需要等待太久就能感受到成长

3. **毒雾跟随玩家而非固定位置**：
   - 固定位置毒雾（类似地雷/陷阱）需要玩家有预判能力，与自动攻击的休闲定位冲突
   - 跟随玩家让毒雾成为"被动光环"，与其他武器同时使用无额外操作负担
   - 但与冰冻光环的差异化：毒雾是纯伤害（无控制），冰冻是纯控场（低伤害）

4. **进化选择圣经和磁铁**：
   - 圣经+毒雾 = 腐蚀领域：圣经的旋转范围特性 + 毒雾的DOT = "旋转毒球+大范围毒"的极致AOE。这是"全屏融化"Build的核心
   - 磁铁+毒雾 = 瘟疫磁力：磁铁的拾取范围 + 毒雾的范围 = "站着不动就能杀敌+收宝石"的站桩Build。这是"懒人最优"Build
   - 两种进化分别满足不同玩家需求：腐蚀领域追求极致伤害，瘟疫磁力追求操作舒适

5. **毒与火是独立DOT系统**：
   - 允许火毒双叠加（火焰法杖+毒雾），创造"双重DOT"的高阶Build
   - 不设计"毒+火=爆炸"的联动，避免强制要求同时携带两个武器才能触发

6. **3个Synergy而非更多**：
   - 护甲+毒雾（猛毒护甲）：防御增强，与毒雾的"站桩"定位匹配
   - 再生+毒雾（腐蚀再生）：续航增强，与站桩流互补
   - 幸运硬币+毒雾（剧毒炼金）：经济增强，利用毒雾的高击杀频率
   - 未选择疾风靴+毒雾（移动速度与范围无关联）、暴击戒指+毒雾（DOT不适合暴击）、生命结晶+毒雾（已有圣水膨胀的半径加成，重复）

7. **魔法师初始武器加入毒雾**：
   - 魔法师从圣水/飞刀/闪电/回旋镖扩展为圣水/飞刀/闪电/回旋镖/毒雾
   - 毒雾作为初始武器提供"开局就能站桩"的体验，与其他初始武器的"主动出击"形成对比

8. **腐蚀领域的毒球DPS 12（3球x4）看似很高**：
   - 但毒球需要敌人碰触才能造成伤害（非全范围覆盖），实际命中率约40-60%
   - 平均毒球DPS约5-7，加上满叠层12.5 = 总计17-20，与FireKnife(25)和Thunderang(24)仍在合理范围
   - 腐蚀领域是"大范围慢杀"型，FireKnife是"高频快杀"型，各有适用场景
