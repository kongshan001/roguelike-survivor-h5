# Pet/伙伴系统完整设计规格

**版本**: v2.4 设计规格
**日期**: 2026-04-06
**Drive**: #29
**来源**: 竞品调研 Spirit Hunters: Infinite Horde 的 Pet 系统 + 设计积压 P3 事项

---

## 1. 设计概述

### 1.1 问题陈述

当前游戏 Build 维度完全依赖"武器+被动+协同"三层，玩家在第 5-6 次升级后 Build 方向基本确定，后续升级只是数值提升。缺乏一个独立的、有自身 AI 行为的战斗维度。竞品 Spirit Hunters 的 Pet 系统证明"跟随型战斗伙伴"能有效增加 Build 多样性和视觉丰富度。

### 1.2 核心设计

Pet 是通过升级面板获取的独立战斗单位，跟随玩家移动，有独特 AI 行为和攻击模式。每种 Pet 有独立的升级路线（3级），不与武器/被动竞争升级槽位，而是作为第四层 Build 维度。

### 1.3 设计约束

- Pet 不占用武器槽或被动槽，是独立的第四层系统
- 最多同时携带 1 只 Pet（避免场上实体过多影响性能）
- Pet 不可被敌人攻击（简化 AI，减少复杂度）
- Pet 的攻击力基于玩家武器伤害乘数（不独立缩放）
- Pet 视觉尺寸小（8-10px），不遮挡游戏画面

## 3. Pet 升级系统

### 3.1 升级机制

Pet 通过升级面板获取和升级，但**不与武器/被动竞争同一升级池**。

**方案**：每 3 次普通升级后，第 4 次升级面板变为"Pet 专属面板"（类似 VS 的宝箱进化时机）。

| 升级次数 | 面板类型 | 可选项 |
|---------|---------|--------|
| 1st-3rd | 标准面板 | 武器/被动/升级 |
| 4th | Pet 面板 | 获取新 Pet（如未拥有）或 Pet 升级（如已拥有） |
| 5th-7th | 标准面板 | 武器/被动/升级 |
| 8th | Pet 面板 | Pet 升级 |
| ... | 每 3 次标准 + 1 次 Pet | 循环 |

**替代方案（更简单，推荐）**：Pet 作为"第 13 种升级选项"加入标准升级池，权重独立。

```
升级池：
- 新武器（未拥有）
- 武器升级（未满级）
- 被动（未满叠）
- 进化（条件满足时）
- Pet 获取（未拥有 Pet 时，权重 2）
- Pet 升级（已拥有 Pet 且未满级时，权重 3）
```

**推荐方案的理由**：
1. 不需要新的 UI 面板或升级节奏变更
2. Pet 升级与武器/被动竞争选择，增加决策深度
3. 权重系统控制 Pet 出现频率（不是每次都出现）
4. 5 分钟短局中 Pet 升级次数约 2-3 次（获取 + 1-2 次升级），不会过度分散注意力

### 3.2 Pet 升级权重

```js
// 在 upgrade-generate.js 的 pool 构建逻辑中:
if (!player.pet && player.level >= 3) {
  pool.push({ type: 'pet_get', weight: 2 });  // Lv3 后开始出现 Pet 选项
}
if (player.pet && player.pet.level < 3) {
  pool.push({ type: 'pet_upgrade', weight: 3 });  // 已有 Pet 时升级优先
}
```

**Lv3 后出现**：让玩家先获得第 1 把武器和 1 个被动，再考虑 Pet。避免 Lv1 就面临 Pet vs 武器的决策。

## 5. Pet AI 与行为系统

### 5.1 跟随 AI

所有 Pet 共用跟随逻辑：

```js
// Pet 跟随逻辑（每帧）:
// 1. 计算目标位置 = 玩家位置 + followAngle方向 * followDist
// 2. 平滑移动向目标位置 (lerp, factor = followSmooth)
// 3. 保持在玩家一定范围内（不超过 150px）
```

| Pet | followAngle | followDist | 说明 |
|-----|-----------|-----------|------|
| 火焰精灵 | -0.5 rad (~-30度) | 40px | 玩家右前方 |
| 冰霜守护 | 2.5 rad (~145度) | 60px | 玩家左后方 |
| 闪电猎鹰 | -1.57 rad (~-90度) | 80px | 玩家头顶（环绕） |

闪电猎鹰特殊：不固定跟随角度，而是在玩家头顶做半径 50px 的环绕运动（orbit.speed 控制角速度）。

### 5.2 攻击 AI

**通用逻辑**：
1. 每帧检查攻击冷却
2. 冷却结束 -> 找最近敌人（300px 范围内）
3. 执行对应攻击模式
4. 重置冷却

**火焰精灵攻击流程**：
```
1. 索敌：最近敌人，200px 范围
2. 发射火球：方向 = pet -> enemy，speed=200
3. 火球碰撞检测：与所有敌人检测
4. 命中：造成 dmg * weaponDmgMul，Lv2+ 附加点燃
5. 穿透：命中后不消失（pierce 次数内），继续飞行
6. 火球超出射程或 pierce 耗尽 -> 移除
```

**冰霜守护攻击流程**：
```
1. 索敌：最近敌人，range 范围内
2. 释放锥形冰霜：以 pet->player.facingAngle 为中轴，张角 0.8rad
3. 范围内敌人：减速 + 伤害
4. Lv3：5% 概率冰冻
5. 视觉：冰蓝色粒子扩散
```

**闪电猎鹰攻击流程**：
```
1. 索敌：最近敌人，range(300px) 范围内
2. 俯冲：pet -> enemy 直线飞去，diveSpeed
3. 命中：造成 dmg * weaponDmgMul
4. 折返：enemy -> player 方向飞回，returnSpeed
5. 回到跟随位置后进入冷却
6. Lv3：命中时 30% 概率触发闪电链
```

## 6. Pet 与现有系统的交互

### 6.1 与武器伤害乘数的交互

Pet 伤害受 `game.weaponDmgMul` 影响，与武器一致：

```
Pet 实际伤害 = baseDmg * (game.weaponDmgMul || 1)
```

这意味着商店"武器精通"升级也提升 Pet 伤害，保持一致性。

### 6.2 与商店系统的交互

Pet 不直接受商店升级影响，但间接受益：
- 武器精通(weaponDmgMul) -> 提升 Pet 伤害
- 拾取精通(pickupRange) -> 不影响 Pet
- 知识汲取(expBonus) -> 不影响 Pet

### 6.3 与协同系统的交互

Pet 暂不参与 Synergy 协同（v2.4 首版简化）。未来 v2.5 可新增 Pet 专属协同：

| 未来协同 | 条件 | 效果（草案） |
|---------|------|------------|
| 火焰共生 | 火焰精灵 + 火焰法杖 | 火球点燃持续+1s |
| 冰霜共鸣 | 冰霜守护 + 冰冻光环 | 冰锥范围+30px |
| 雷鹰风暴 | 闪电猎鹰 + 闪电 | 俯冲触发闪电链概率+20% |

### 6.4 与连击系统的交互

Pet 击杀计入玩家连击计数。Pet 攻击频率适中（最快0.6s），不会刷连击。

### 6.5 与食物/宝石系统的交互

Pet 击杀的敌人正常掉落宝石和食物，计入玩家统计。

### 6.6 与成就系统的交互

新增 2 个 Pet 相关成就：

```js
// CFG.ACHIEVEMENTS 新增

  name: '忠实的伙伴', icon: '🐾', desc: '首次获得一只Pet',
  type: 'flag', reward: 30

  name: '完美搭档', icon: '⭐', desc: '将一只Pet升至满级',

```

### 6.7 与 Quest 系统的交互

新增 1 个 Pet 相关 Quest：

```js
// CFG.QUESTS 新增
{ id:'pet_kill_30', name:'宠物猎人', icon:'🐾', desc:'Pet击杀30个敌人',
  check: s => s.petKills >= 30, reward: 80 }
```

### 6.8 与无尽模式的交互

Pet 在无尽模式中持续跟随，伤害受无尽难度影响（敌人 HP 递增但 Pet 伤害不递增），后期 Pet 更多作为辅助而非主力。

| 冰冻光环 | ~1 | ~2 |

**结论**：
- 火焰精灵 Lv3 DPS(10.3) 接近基础武器 Lv3 水平，但需要牺牲一次升级选择
- 冰霜守护 DPS 极低(2.0@Lv3)，但提供减速/冰冻控制，价值不在伤害
- 闪电猎鹰 Lv3 DPS(8.3) 主要集中在单次高伤，类似回旋镖定位
- Pet 不替代武器，而是"锦上添花"的第四维度

### 10.2 5 分钟局 Pet 升级节奏

```
Lv3（~0:50）: Pet 获取选项开始出现在升级池
Lv4-5（~1:00-1:40）: 大多数玩家获取 Pet
Lv7-8（~2:30-3:00）: Pet 升级到 Lv2
Lv10-11（~3:45-4:10）: Pet 可能升到 Lv3（如果有幸选中）
```

5 分钟局内 Pet 大约获得 2-3 次升级选择（不是每次都选 Pet），实际升级 1-2 次。Lv3 Pet 需要大约 4 分钟，在 Boss 战前可以满级。

### 10.3 性能影响

| 维度 | 影响 | 说明 |
|------|------|------|
| 实体数 | +1 Pet + 0-5 火球 | 极小，不影响 MAX_ENEMIES |
| 碰撞检测 | Pet 攻击需要碰撞检测 | 复用现有碰撞框架 |
| 绘制 | +1 精灵 + 少量粒子 | 极小，单次 draw call |
| AI 计算 | +1 跟随 + 1 攻击 | 极小 |

## 11. 设计决策记录

1. **最多 1 只 Pet 而非更多**：5 分钟短局 + Canvas 2D 性能限制。多只 Pet 会让场上实体过多，且升级选择分散。1 只 Pet 的"伙伴感"更强。

2. **Pet 不可被攻击**：简化 AI 逻辑。如果 Pet 可被攻击，需要 HP 管理、复活机制、无敌帧等大量额外系统。Pet 作为"额外能力"而非"需要保护的脆弱单位"更符合 5 分钟快节奏。

3. **Pet 使用标准升级池而非独立面板**：避免新的 UI 系统和升级节奏打断。Pet 作为升级选项之一增加决策深度（"拿武器还是拿 Pet？"）。权重系统控制出现频率。

4. **Lv3 后才出现 Pet 选项**：让玩家先确立武器方向再考虑 Pet。Lv1 就面对 Pet vs 武器的选择会造成新手困惑。

5. **3 种 Pet 覆盖投射/控场/高伤三角**：与武器设计哲学一致。火焰精灵=投射输出、冰霜守护=控场辅助、闪电猎鹰=高伤单体。玩家可以根据已有 Build 选择互补的 Pet。

6. **Pet 伤害受 weaponDmgMul 影响**：保持一致性。商店"武器精通"升级让所有伤害来源受益，包括 Pet。避免 Pet 成为独立缩放的伤害维度。

7. **Pet 暂不参与 Synergy 协同**：v2.4 首版简化实现。Pet 系统本身是新系统，先验证基础体验再增加复杂度。未来协同方向已在草案中定义。

8. **火焰精灵 Lv3 点燃 1.5s 而非更长**：避免与火焰法杖的点燃(2s)完全重叠。Pet 的点燃是"小火"，弱于武器"大火"。

9. **闪电猎鹰用环绕飞行而非固定跟随**：视觉辨识度。环绕飞行让猎鹰看起来像"空中巡逻"，俯冲+折返的攻击路径创造独特的节奏感。

10. **冰霜守护与冰冻光环数值同级**：让控场 Build 可以"双重减速"（光环 + Pet 叠加），但单一来源的控场已经足够。不设计成 Pet 弱于武器（否则没人选 Pet）。

11. **新增 Pet 击杀统计**：为 Quest 和成就提供数据支持。Pet 击杀也计入玩家总击杀和连击，不独立统计 KDA。

12. **Pet 不掉落灵魂碎片或金币**：Pet 击杀的敌人正常参与经济系统，但 Pet 本身不产生额外经济收益，避免"Pet = 金矿"的优化路径。

## 12. 未来扩展方向

### v2.5 Pet 协同系统

3 个 Pet x 3 个对应武器 = 3 个 Pet+武器协同：
- 火焰精灵 + 火焰法杖 -> 火焰共生（点燃+1s）
- 冰霜守护 + 冰冻光环 -> 冰霜共鸣（范围+30px）
- 闪电猎鹰 + 闪电 -> 雷鹰风暴（闪电链概率+20%）

### v2.6 进化 Pet

Pet 满级(Lv3) + 特定被动满叠 -> 进化 Pet：
- 火焰精灵 + 暴击戒指 -> 烈焰凤凰（俯冲AOE + 暴击）
- 冰霜守护 + 护甲 -> 寒冰壁垒（全方位护盾 + 冰冻）
- 闪电猎鹰 + 磁铁 -> 磁暴鹰（俯冲吸引宝石 + 闪电链）

### v2.7 更多 Pet

新增 2-3 种 Pet（如治疗精灵=范围回复、暗影蝙蝠=吸血攻击），扩展 Pet 选择深度。
