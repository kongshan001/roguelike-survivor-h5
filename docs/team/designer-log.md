## 2026-04-03 — 策划迭代12：音效系统设计

### 背景
游戏目前完全无声，缺乏打击反馈和氛围感。音效系统使用 Web Audio API 振荡器合成 8-bit 风格音效，零外部文件，保持单 HTML 文件架构。

### 成果

**SFX 配置表**：

| ID | 触发时机 | 音色 | 频率 | 时长 | 波形 | 说明 |
|----|---------|------|------|------|------|------|
| `hit` | 玩家受伤 | 下行短音 | 440→110Hz | 0.15s | square | 受击反馈 |
| `kill` | 敌人死亡 | 上行短音 | 200→600Hz | 0.1s | square | 击杀爽感 |
| `knife` | 飞刀发射 | 短促刺音 | 800Hz | 0.05s | sawtooth | 投掷感 |
| `lightning` | 闪电击中 | 噪声爆发 | 白噪声 | 0.12s | 白噪声 | 电击感 |
| `levelup` | 升级 | 三个上行音阶 | C5-E5-G5 | 0.3s | square | 成就感 |
| `pickup` | 拾取宝石/食物 | 短促叮声 | 880Hz | 0.08s | sine | 收集感 |
| `chest` | 开箱 | 上行琶音 | 440→660→880Hz | 0.25s | triangle | 惊喜感 |
| `boss` | Boss出场 | 低沉警告 | 110Hz×3 | 0.6s | sawtooth | 危机感 |
| `freeze` | 冰冻触发 | 高频碎裂 | 1200→400Hz | 0.15s | sine | 冰碎感 |
| `gameover` | 死亡/失败 | 下行长音 | 440→110Hz | 0.8s | sawtooth | 失败感 |
| `victory` | 通关 | 上行凯旋 | C5→E5→G5→C6 | 0.5s | square | 胜利感 |

**实现方案**：
- `SFX` 全局对象，`init()` 在首次用户交互时创建 `AudioContext`（满足浏览器自动播放策略）
- `play(id)` 方法：根据 ID 查表创建 `OscillatorNode` + `GainNode`，播放后自动断开
- 音量控制：`masterVolume = 0.3`（默认30%音量，避免太吵）
- 白噪声实现：`AudioBuffer` 填充随机采样值

**CONFIG 数值**：
```javascript
SFX: {
  enabled: true,
  volume: 0.3,
  sounds: {
    hit:      { freq:[440,110], dur:0.15, type:'square' },
    kill:     { freq:[200,600], dur:0.10, type:'square' },
    knife:    { freq:800,       dur:0.05, type:'sawtooth' },
    lightning:{ noise:true,     dur:0.12 },
    levelup:  { freq:[523,659,784], dur:0.1, type:'square', seq:true },
    pickup:   { freq:880,       dur:0.08, type:'sine' },
    chest:    { freq:[440,660,880], dur:0.08, type:'triangle', seq:true },
    boss:     { freq:110,       dur:0.6,  type:'sawtooth', repeat:3 },
    freeze:   { freq:[1200,400],dur:0.15, type:'sine' },
    gameover: { freq:[440,110], dur:0.8,  type:'sawtooth' },
    victory:  { freq:[523,659,784,1047], dur:0.12, type:'square', seq:true }
  }
}
```

**触发点**（10个）：
1. `player.takeDamage()` → `SFX.play('hit')`
2. 敌人 `hp<=0` → `SFX.play('kill')`
3. `Knife.update()` 发射时 → `SFX.play('knife')`
4. `Lightning.update()` 电击时 → `SFX.play('lightning')`
5. `player.addExp()` 返回 true → `SFX.play('levelup')`
6. 宝石被收集 → `SFX.play('pickup')`
7. 宝箱开箱 → `SFX.play('chest')`
8. Boss 生成 → `SFX.play('boss')`
9. 冰冻触发 → `SFX.play('freeze')`
10. `endGame()` → `SFX.play('gameover')` 或 `SFX.play('victory')`

### 决策记录
- 8-bit 合成音与像素风视觉风格统一
- 不使用外部音频文件，保持单文件零依赖架构
- `AudioContext` 延迟创建（首次交互时），满足 Chrome 自动播放策略
- 音效数量控制在10个，覆盖核心反馈但不过度
- 飞刀和闪电有专属音效，其他武器（圣水/圣经/火焰/冰冻）使用通用 kill/pickup 音效
- 未来可扩展：BGM 循环、更多武器专属音效

## 2026-04-03 — 策划迭代11：第6种武器 — 冰冻光环

### 背景
现有5种基础武器覆盖：环绕(圣水)、投射(飞刀)、随机(闪电)、范围(圣经)、锥形(火焰法杖)。缺少**控场型**武器。冰冻光环填补"范围减速+冰冻控制"生态位。

### 成果

**冰冻光环 (FrostAura)**：以玩家为中心的范围减速光环，有概率冰冻敌人

| 属性 | Lv1 | Lv2 | Lv3 |
|------|-----|-----|-----|
| 范围半径 | 80px | 100px | 130px |
| 减速比例 | 30% | 45% | 60% |
| 冰冻触发概率 | 0% | 0% | 8%/秒 |
| 冰冻持续 | — | — | 1.5秒 |
| 脉冲伤害 | 1/s | 1.5/s | 2/s |

**CONFIG 数值**：
```javascript
frostaura: {
  name:'冰冻光环', icon:'❄️', desc:'范围减速',
  Lv: {
    1: { radius:80, slow:0.3, dmg:1, freezeChance:0, freezeDur:0 },
    2: { radius:100, slow:0.45, dmg:1.5, freezeChance:0, freezeDur:0 },
    3: { radius:130, slow:0.60, dmg:2, freezeChance:0.08, freezeDur:1.5 }
  }
}
```

**机制**：
- 以玩家为中心的圆形范围
- 范围内敌人移速×(1-slow)，持续受到少量伤害
- Lv3解锁冰冻：每秒8%概率触发，敌人完全停止1.5秒
- 冰冻中的敌人变蓝+闪烁特效

**视觉**：
- 浅蓝色半透明圆环围绕玩家
- 范围内敌人脚下出现冰晶粒子
- 冰冻触发时：敌人变蓝+白色冰晶覆盖
- 脉冲效果：每2秒一圈冰蓝波纹从中心向外扩散

**获取方式**：仅通过升级面板获取（不在初始选择池）

### 决策记录
- 冰冻光环是唯一的纯控场武器，让"走位+控场"打法成为可能
- 低伤害（2/s）是为了平衡减速的强大控场效果
- Lv3才解锁冰冻，与火焰法杖Lv3解锁点燃一致
- 与现有武器互补：减速后飞刀/闪电更容易命中，冰冻后圣经/圣水可以安全输出
- 未来进化路线预留：冰冻光环+闪电→暴风雪（大范围冰冻+闪电链）

### 背景
当前所有游戏开始时角色完全相同（相同属性、相同精灵、从武器三选一开始）。多角色系统增加重玩价值，每个角色有独特定位。

### 设计原则
- MVP只做3个角色，覆盖"均衡/坦克/速攻"三角
- 角色差异体现在初始属性和初始武器，不做专属技能（降低复杂度）
- 选角色在选武器之前，选完角色后自动获得初始武器（跳过武器选择）

### 成果

**3个角色设计**：

| 角色 | 名称 | 初始武器 | HP | 移速 | 特点 |
|------|------|---------|-----|------|------|
| 🧙 魔法师 | 艾伦 | 自选（圣水/飞刀/闪电） | 8 | 160 | 均衡型，现有体验不变 |
| 🛡 战士 | 凯恩 | 飞刀（固定） | 12 | 140 | 高血量坦克，移速较慢 |
| 🏹 游侠 | 希尔 | 圣水（固定） | 6 | 190 | 低血量高速，走位风 |

**CONFIG 数值**：
```javascript
CHARACTERS: {
  mage:   { name:'魔法师', icon:'🧙', hp:8, speed:160, desc:'均衡型，自选初始武器', chooseWeapon:true },
  warrior:{ name:'战士',   icon:'🛡', hp:12,speed:140, desc:'高血量坦克',   startWeapon:'knife' },
  ranger: { name:'游侠',   icon:'🏹', hp:6, speed:190, desc:'高速低血，走位风', startWeapon:'holywater' }
}
```

**流程变更**：
1. 标题画面"开始游戏" → 显示**角色选择**画面（3张角色卡）
2. 选择魔法师 → 进入武器选择（现有3选1流程不变）
3. 选择战士/游侠 → 直接进入游戏（自动装备初始武器）
4. 角色选择界面标题："选择你的英雄"

**角色视觉差异**（由美术定义）：
- 魔法师：现有蓝色法师（不变）
- 战士：红棕色铠甲+大剑
- 游侠：绿色斗篷+弓

### 决策记录
- 3角色覆盖三角：坦克(高HP低速度)、均衡(中HP中速)、速攻(低HP高速)
- 魔法师保持"自选武器"是向后兼容，老玩家体验不变
- 战士固定飞刀：近战角色配自动投掷武器，靠血量硬抗
- 游侠固定圣水：高速角色配环绕防御武器，核心是走位
- 不做被动差异（如战士自带护甲），避免平衡复杂化
- 角色选择界面与武器选择界面共用CSS样式（ws-card/upg-card），减少代码量

| 奖励 | 效果 | 说明 |

- 3种奖励覆盖"续航/机动/成长"三角，玩家不需要刻意选择（随机），增加惊喜感
- 不做UI弹窗选择，靠近自动开箱，保持操作简洁
- 临时加速用 `player._speedBoost={mult,dur}` 实现，不影响被动道具系统

## 2026-04-03 — 策划迭代8：新敌人 — 幽灵 (Ghost)

### 成果
**幽灵敌人设计**：不可预测的闪烁型敌人，穿墙+低血瞬移

| 属性 | 数值 |
|------|------|
| 尺寸 | 12×12（小体型，难瞄准） |
| 血量 | 2 HP（脆弱，但难命中） |
| 移速 | 55px/s（中等，比僵尸快比蝙蝠慢） |
| 伤害 | 1（普通接触伤害） |
| 特殊：闪烁 | 每2秒进入"半透明"状态1秒，期间受到的伤害减半 |
| 特殊：瞬移 | HP=1时触发瞬移，瞬间闪现到玩家背后80-120px处 |
| 特殊：穿墙 | 无视地图边界（如有），直线穿越（目前无限地图无墙，保留字段） |

**CONFIG 数值**：
```javascript
ghost: { w:12, h:12, hp:2, speed:55, dmg:1, color:'#b0bec5', phaseShift:true, teleport:true }
```

**生成时机**：
- 2分钟（120s）后开始混入生成池
- 出现在 `getSpawnRate()` 的 180s+ 阶段，与僵尸/蝙蝠/骷髅混编
- 生成权重低于僵尸，高于骷髅（不常出现，但每次出现都有威胁感）

**行为设计**：
1. **追踪**：像僵尸一样追踪玩家，但速度更快
2. **闪烁（Phase Shift）**：每2秒进入1秒"半透明"状态
   - 视觉：alpha在0.2-0.5间脉动，给玩家"快打我/等一下"的节奏提示
   - 机制：半透明期间受到的伤害×0.5（向下取整，1伤害→0伤害）
   - 战术：玩家需要看准实体状态时攻击
3. **瞬移（Teleport）**：HP降至1时触发一次
   - 闪现到玩家背后80-120px处（反方向随机偏移±30°）
   - 瞬移后有0.5秒实体化延迟（不造成伤害），给玩家反应时间
   - 每只幽灵只触发一次瞬移

**食物掉落**：
- 幽灵掉落：🍞 面包（白色 `#e0e0e0`），与现有食物系统共用拾取逻辑

### 决策记录
- 幽灵填补"不可预测"生态位——现有4种敌人行为都是确定性追踪，幽灵通过闪烁+瞬移打破节奏
- HP=2配合闪烁减伤，实际需要2-3次命中才能击杀（闪烁期间打不疼），延长了交战时间
- 瞬移只在HP=1时触发一次，避免反复瞬移让玩家烦躁
- 生成在2分钟后混入，此时玩家已有1-2种武器，有能力应对闪烁机制
- 尺寸12×12比蝙蝠略大，保证在移动端也能看清闪烁状态

## 2026-04-03 — 策划迭代6：第二条进化路线 — 火焰飞刀

### 成果
**进化路线**：飞刀 Lv.3 + 火焰法杖 Lv.3 → 🔥🗡 火焰飞刀

| 属性 | 数值 |
|------|------|
| 投掷数量 | 5把（继承飞刀Lv3） |
| 飞行速度 | 280px/s（略快于普通飞刀250） |
| 命中伤害 | 3（与飞刀Lv3相同） |
| 穿透 | 2次（飞刀Lv3只有1次） |
| 燃烧DOT | 3/s持续2s（继承火焰法杖Lv3点燃） |
| 投掷间隔 | 0.5s（快于普通飞刀0.7s） |

**CONFIG**：
```javascript
{ a:'knife', b:'firestaff', result:'fireknife', name:'火焰飞刀', icon:'🔥🗡', desc:'燃烧穿透飞刀' }
```

**机制**：
- 继承飞刀的自动瞄准最近敌人+投掷

## 2026-04-03 — 策划迭代13：第4条进化路线 — 暴风雪

### 背景
冰冻光环(v0.8.0)加入后没有对应的进化路线，是唯一没有进化的基础武器。第4条进化路线让冰冻光环+闪电满级→暴风雪，完成进化系统四角覆盖。

### 成果

**进化路线**：冰冻光环(frostaura) Lv.3 + 闪电(lightning) Lv.3 → ❄️⚡ 暴风雪(Blizzard)

| 属性 | 数值 |
|------|------|
| 范围半径 | 160px（大于冰冻光环Lv3的130px） |
| 减速比例 | 70%（强于冰冻光环Lv3的60%） |
| 冰冻触发 | 15%/秒（强于冰冻光环Lv3的8%），持续2秒 |
| 持续伤害 | 3/s（冰冻光环2/s + 闪电1/s 合计） |
| 闪电链 | 每2秒对范围内随机3个敌人释放链式闪电（伤害8，2链，50%衰减） |
| 冰晶风暴 | 每3秒释放一圈冰晶弹幕（12方向，向外扩散80px，伤害4） |

**CONFIG**：
```javascript
{ a:'frostaura', b:'lightning', result:'blizzard', name:'暴风雪', icon:'❄️⚡', desc:'大范围暴风雪+闪电链' }
```

**机制**：
- 继承冰冻光环的范围减速+冰冻机制（扩大至160px）
- 范围内持续3/s伤害
- 每2秒触发闪电链：选3个随机敌人，伤害8，2链递减50%
- 每3秒释放12方向冰晶弹幕：向外扩散80px后消失，每个冰晶伤害4
- 进化光环：冰蓝色+黄色电光交替脉动

**升级路线**：
- 进化武器固定为Lv1（不可升级，maxLevel=1）

### 决策记录
- 暴风雪填补"大范围控场+AOE"生态位
- 闪电链保留闪电武器的核心体验（链式电击）
- 冰晶弹幕增加视觉丰富度，让武器"感觉强力"
- 160px范围配合70%减速=几乎无法逃脱的控场区
- 伤害3/s偏低（纯DPS不如其他进化武器），但闪电链+冰晶弹幕提供了额外爆发伤害
- 与现有3条进化路线形成互补：雷暴圣水(持续+链)、火焰飞刀(穿透+燃烧)、圣光领域(大范围+脉冲)、暴风雪(控场+闪电)
