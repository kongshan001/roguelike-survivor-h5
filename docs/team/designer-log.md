## 2026-04-03 — 策划迭代10：多角色系统

### 背景
当前所有游戏开始时角色完全相同（相同属性、相同精灵、从武器三选一开始）。多角色系统增加重玩价值，每个角色有独特定位。

### 设计原则
- MVP只做3个角色，覆盖"均衡/坦克/速攻"三角
- 角色差异体现在初始属性和初始武器，不做专属技能（降低复杂度）
- 选角色在选武器之前，选完角色后自动获得初始武器（跳过武器选择）

### 成果

**3个角色设计**：

| 角色 | 名称 | 初始武器 | HP | 移速 | 特点 |
|------|------|---------|-----|------|------|
| 🧙 魔法师 | 艾伦 | 自选（圣水/飞刀/闪电） | 8 | 160 | 均衡型，现有体验不变 |
| 🛡 战士 | 凯恩 | 飞刀（固定） | 12 | 140 | 高血量坦克，移速较慢 |
| 🏹 游侠 | 希尔 | 圣水（固定） | 6 | 190 | 低血量高速，走位风 |

**CONFIG 数值**：
```javascript
CHARACTERS: {
  mage:   { name:'魔法师', icon:'🧙', hp:8, speed:160, desc:'均衡型，自选初始武器', chooseWeapon:true },
  warrior:{ name:'战士',   icon:'🛡', hp:12,speed:140, desc:'高血量坦克',   startWeapon:'knife' },
  ranger: { name:'游侠',   icon:'🏹', hp:6, speed:190, desc:'高速低血，走位风', startWeapon:'holywater' }
}
```

**流程变更**：
1. 标题画面"开始游戏" → 显示**角色选择**画面（3张角色卡）
2. 选择魔法师 → 进入武器选择（现有3选1流程不变）
3. 选择战士/游侠 → 直接进入游戏（自动装备初始武器）
4. 角色选择界面标题："选择你的英雄"

**角色视觉差异**（由美术定义）：
- 魔法师：现有蓝色法师（不变）
- 战士：红棕色铠甲+大剑
- 游侠：绿色斗篷+弓

### 决策记录
- 3角色覆盖三角：坦克(高HP低速度)、均衡(中HP中速)、速攻(低HP高速)
- 魔法师保持"自选武器"是向后兼容，老玩家体验不变
- 战士固定飞刀：近战角色配自动投掷武器，靠血量硬抗
- 游侠固定圣水：高速角色配环绕防御武器，核心是走位
- 不做被动差异（如战士自带护甲），避免平衡复杂化
- 角色选择界面与武器选择界面共用CSS样式（ws-card/upg-card），减少代码量

## 2026-04-03 — 策划迭代9：宝箱系统 — 金币消耗途径

### 背景
BUG-007：结算画面金币用途不明确。金币目前仅击杀累积+结算展示，无消耗途径。
宝箱系统让金币成为有意义的资源，增加决策深度。

### 成果

**宝箱机制设计**：

| 属性 | 数值 |
|------|------|
| 宝箱价格 | 20金币/次 |
| 生成方式 | 地图上固定位置刷新，每90秒刷新1个，同屏最多2个 |
| 生成位置 | 玩家周围300-500px随机位置 |
| 宝箱外观 | 金色箱子（20×16px） |
| 开启方式 | 靠近后自动检测（35px拾取范围），弹出"花20金币开箱？"提示 |

**宝箱奖励池**（等概率随机3选1）：

| 奖励 | 效果 | 说明 |
|------|------|------|
| 💊 回复药水 | HP+3 | 即时回血，满血不浪费（上限8） |
| ⚡ 临时加速 | 移速+50%持续10秒 | 紧急逃脱/快速收割 |
| 💎 经验宝石 | 20经验值 | 直接获得经验，加速升级 |

**CONFIG 数值**：
```javascript
CHEST: {
  cost: 20,
  spawnInterval: 90,
  maxChests: 2,
  spawnRange: [300, 500],
  pickupRange: 35,
  rewards: [
    { type:'heal', icon:'💊', value:3, desc:'回复3HP' },
    { type:'speed', icon:'⚡', value:0.5, duration:10, desc:'移速+50% 10秒' },
    { type:'exp', icon:'💎', value:20, desc:'+20经验' }
  ]
}
```

**交互流程**：
1. 玩家靠近宝箱 → 显示"开箱 20💰"浮动文字
2. 自动开箱（有足够金币时），金币-20
3. 宝箱消失，弹出奖励特效（icon上浮+描述文字）
4. 金币不足时显示"金币不足"红色文字

### 决策记录
- 宝箱解决金币用途问题的同时，增加地图探索动力（不只是原地刷怪）
- 20金币/次 = 平均击杀2只怪的代价，适中但不过分
- 3种奖励覆盖"续航/机动/成长"三角，玩家不需要刻意选择（随机），增加惊喜感
- 不做UI弹窗选择，靠近自动开箱，保持操作简洁
- 临时加速用 `player._speedBoost={mult,dur}` 实现，不影响被动道具系统

## 2026-04-03 — 策划迭代8：新敌人 — 幽灵 (Ghost)

### 成果
**幽灵敌人设计**：不可预测的闪烁型敌人，穿墙+低血瞬移

| 属性 | 数值 |
|------|------|
| 尺寸 | 12×12（小体型，难瞄准） |
| 血量 | 2 HP（脆弱，但难命中） |
| 移速 | 55px/s（中等，比僵尸快比蝙蝠慢） |
| 伤害 | 1（普通接触伤害） |
| 特殊：闪烁 | 每2秒进入"半透明"状态1秒，期间受到的伤害减半 |
| 特殊：瞬移 | HP=1时触发瞬移，瞬间闪现到玩家背后80-120px处 |
| 特殊：穿墙 | 无视地图边界（如有），直线穿越（目前无限地图无墙，保留字段） |

**CONFIG 数值**：
```javascript
ghost: { w:12, h:12, hp:2, speed:55, dmg:1, color:'#b0bec5', phaseShift:true, teleport:true }
```

**生成时机**：
- 2分钟（120s）后开始混入生成池
- 出现在 `getSpawnRate()` 的 180s+ 阶段，与僵尸/蝙蝠/骷髅混编
- 生成权重低于僵尸，高于骷髅（不常出现，但每次出现都有威胁感）

**行为设计**：
1. **追踪**：像僵尸一样追踪玩家，但速度更快
2. **闪烁（Phase Shift）**：每2秒进入1秒"半透明"状态
   - 视觉：alpha在0.2-0.5间脉动，给玩家"快打我/等一下"的节奏提示
   - 机制：半透明期间受到的伤害×0.5（向下取整，1伤害→0伤害）
   - 战术：玩家需要看准实体状态时攻击
3. **瞬移（Teleport）**：HP降至1时触发一次
   - 闪现到玩家背后80-120px处（反方向随机偏移±30°）
   - 瞬移后有0.5秒实体化延迟（不造成伤害），给玩家反应时间
   - 每只幽灵只触发一次瞬移

**食物掉落**：
- 幽灵掉落：🍞 面包（白色 `#e0e0e0`），与现有食物系统共用拾取逻辑

### 决策记录
- 幽灵填补"不可预测"生态位——现有4种敌人行为都是确定性追踪，幽灵通过闪烁+瞬移打破节奏
- HP=2配合闪烁减伤，实际需要2-3次命中才能击杀（闪烁期间打不疼），延长了交战时间
- 瞬移只在HP=1时触发一次，避免反复瞬移让玩家烦躁
- 生成在2分钟后混入，此时玩家已有1-2种武器，有能力应对闪烁机制
- 尺寸12×12比蝙蝠略大，保证在移动端也能看清闪烁状态

## 2026-04-03 — 策划迭代7：第三条进化路线 — 圣光领域

### 成果
**进化路线**：圣经(bible) Lv.3 + 圣水(holywater) Lv.3 → 📖💧 圣光领域

| 属性 | 数值 |
|------|------|
| 范围半径 | 130px（大于圣经Lv3的120px） |
| 旋转速度 | 4 rad/s |
| 伤害 | 2.5/s 持续（圣水伤害+圣经范围结合） |
| 圣光爆发 | 每4秒释放一次全屏圣光脉冲，200px范围内所有敌人受12伤害 |
| 球体数量 | 4个（圣水3球+1个额外） |

**CONFIG**：
```javascript
{ a:'bible', b:'holywater', result:'holydomain', name:'圣光领域', icon:'📖💧', desc:'超大范围+圣光脉冲' }
```

**机制**：
- 继承圣经的范围旋转机制（扩大至130px半径）
- 4个圣光球体在范围内旋转，接触持续伤害
- 每4秒爆发一次圣光脉冲（视觉：白色圆形扩散），200px范围AOE伤害12
- 双重伤害源：球体持续接触 + 脉冲AOE

### 决策记录
- 圣光领域填补"超大范围清怪"生态位，与雷暴圣水（链式闪电）和火焰飞刀（穿透燃烧）形成三角
- 4秒脉冲给玩家清晰的节奏感，不是纯被动
- 伤害12的脉冲相当于秒杀大多数普通敌人，但4秒间隔限制了总DPS

## 2026-04-03 — 策划迭代6：第二条进化路线 — 火焰飞刀

### 成果
**进化路线**：飞刀 Lv.3 + 火焰法杖 Lv.3 → 🔥🗡 火焰飞刀

| 属性 | 数值 |
|------|------|
| 投掷数量 | 5把（继承飞刀Lv3） |
| 飞行速度 | 280px/s（略快于普通飞刀250） |
| 命中伤害 | 3（与飞刀Lv3相同） |
| 穿透 | 2次（飞刀Lv3只有1次） |
| 燃烧DOT | 3/s持续2s（继承火焰法杖Lv3点燃） |
| 投掷间隔 | 0.5s（快于普通飞刀0.7s） |

**CONFIG**：
```javascript
{ a:'knife', b:'firestaff', result:'fireknife', name:'火焰飞刀', icon:'🔥🗡', desc:'燃烧穿透飞刀' }
```

**机制**：
- 继承飞刀的自动瞄准最近敌人+投掷
- 每把飞刀命中后附加燃烧效果（`enemy._burn={dmg:3,t:2}`）
- 燃烧叠加（多次命中重置持续时间）
- 穿透2次：穿过前2个敌人后消失
- 视觉：橙红色飞刀 + 命中爆火花

### 决策记录
- 火焰飞刀填补"高频多目标+持续伤害"的生态位
- 穿透+燃烧的组合让清怪效率显著提升，是进化武器的应有强度
- 与雷暴圣水（范围持续）形成互补

## 2026-04-03 — 策划迭代5：地图道具拾取 — 食物回血

### 成果
**食物回血系统设计**：

| 属性 | 数值 |
|------|------|
| 掉落概率 | 10%（每只敌人死亡） |
| 回复量 | 1 HP（1颗❤️） |
| 拾取范围 | 与宝石相同（35px快速吸入，全图漂移） |
| 存活时间 | 15秒后消失 |
| 上限 | 同屏最多8个食物 |
| Boss掉落 | 必掉3个食物 |

**食物类型**（纯视觉区分，效果相同）：
- 🍖 烤肉（僵尸掉落）
- 🍇 葡萄（蝙蝠掉落）
- 🧀 奶酪（骷髅掉落）

**CONFIG 数值**：
```javascript
FOOD: {
  dropRate: 0.1,       // 10% drop chance
  healAmount: 1,       // 1 HP
  maxFood: 8,          // max on screen
  lifetime: 15,        // seconds before despawn
  bossDropCount: 3     // boss guaranteed drops
}
```

**实现要点**：
- Food 类似 Gem 类，有位置/漂移/拾取逻辑
- 拾取时 `player.hp = min(player.hp + 1, player.maxHp)`
- 绘制：对应 emoji 或像素食物精灵
- 与 Gem 共用全图漂移机制

### 决策记录
- 食物回血是生存类游戏的基础机制，玩家HP上限8但无自然恢复，食物提供额外续航
- 10%掉落率意味着平均每10只怪掉1个食物，配合后期密集刷怪提供稳定回复
- Boss必掉3个食物，作为战斗中的"补给站"
- 回复量仅1HP，避免食物过强导致无敌
